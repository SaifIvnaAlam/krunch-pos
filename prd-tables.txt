=== TABLE 16 ===
STEP 1 ΓÇö Terminal detects OFFLINE_MODE (3s WebSocket timeout)
Status bar shows amber offline indicator

STEP 2 ΓÇö Staff enters PIN on login screen

STEP 3 ΓÇö Terminal looks up staff in local SQLite offline_auth_cache:
SELECT * FROM offline_auth_cache WHERE staffId = ?

STEP 4 ΓÇö Verify PIN locally:
a. Compute PBKDF2(enteredPin, storedSalt, 100000 iterations, SHA-256)
b. Compare to stored PBKDF2 hash in SQLite
c. If match: PASS. If no match: increment failCount.

STEP 5 ΓÇö If failCount >= 5:
a. Lock terminal for this staff member for 60 seconds
b. Log failed attempt in offline_audit_log

STEP 6 ΓÇö If PIN verified:
a. Decode cached JWT (no network call needed ΓÇö just decode the payload)
b. Check cached roles and permissions
c. If cachedToken.exp < now AND > (now - 8hr grace):
Allow login with cached permissions (grace period)
d. If cachedToken.exp < (now - 8hr):
Reject ΓÇö token too stale, manager PIN required to unlock

STEP 7 ΓÇö All actions taken offline are tagged: { offlineAuth: true }
These are included in the offline_log and synced on reconnection

STEP 8 ΓÇö On reconnect: full online auth flow runs silently in background
Fresh tokens replace cached tokens

=== TABLE 19 ===
STAFF LOGOUT (voluntary):
1. Staff taps "Log Out" button
2. Terminal clears accessToken from memory
3. Terminal sends POST /api/v1/auth/logout { refreshToken }
4. Server adds refreshToken to Redis blacklist (TTL = remaining expiry)
5. Terminal returns to login screen

SHIFT END (automatic):
1. Shift end time reached (from staff schedule)
2. Terminal triggers automatic logout flow above
3. Any open orders are flagged for handover

FORCE LOGOUT (manager action from admin dashboard):
1. Manager clicks "Force Logout" on staff member in admin
2. Server blacklists all active tokens for that staffId
3. Terminal receives logout push via WebSocket sync gateway
4. Terminal clears session immediately

=== TABLE 26 ===
RBAC GUARD ΓÇö runs on every API request
ΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇΓöÇ
1.  Extract JWT from Authorization header
2.  Verify JWT signature (using public key)
3.  Check JWT is not in Redis blacklist (logged out / force revoked)
4.  Extract { staffId, branchId, roles[], permissions[] } from payload
5.  Extract required permission from route decorator:
@RequirePermission('orders:void')
6.  Check branch scope:
Is the requested resource in the same branchId as the token?
If branchId mismatch AND role is not OWNER/ADMIN/AUDITOR ΓåÆ 403
7.  Check permission:
Is 'orders:void' in token.permissions[]? ΓåÆ ALLOW
Is it not? ΓåÆ 403 Forbidden

static merge(a: VectorClock, b: VectorClock): VectorClock {
const merged: VectorClock = {};
const allKeys = new Set([...Object.keys(a), ...Object.keys(b)]);
for (const key of allKeys) {
merged[key] = Math.max(a[key] ?? 0, b[key] ?? 0);
}
return merged;
}
